# 🏋️ AI 健身管家

一个基于 **微信小程序 + FastAPI + 大模型（Qwen）** 的 AI 健身助手，  
为用户提供个性化的训练计划、饮食建议以及智能健身问答。

---

## 🚀 项目简介

本项目旨在打造一个“像私人教练一样”的 AI 健身助手，支持：

- 📊 体脂计算与基础代谢分析
- 🥗 AI 个性化饮食推荐（含食物清单）
- 💪 AI 训练计划生成（含动作级别细节）
- 🤖 AI 聊天教练（多轮对话 + 用户画像）
- 🔗 聊天与功能联动（推荐直接跳转训练/饮食页面）

---

## 🧩 核心功能

### 1. 体脂与代谢计算
- 根据用户输入（身高、体重、年龄、性别）
- 自动计算 BMR / TDEE / 热量目标

---

### 2. AI 饮食推荐（结构化输出）
- 自动计算三大营养素（蛋白质 / 脂肪 / 碳水）
- AI 生成每日饮食方案：
  - 早餐 / 午餐 / 晚餐 / 加餐
- 提供可执行的食物清单（非泛化建议）

---

### 3. AI 训练计划生成（动作级别）
- 根据目标 + 经验 + 训练天数生成完整计划
- 支持：
  - 每日训练部位拆分
  - 动作列表（4~6个动作/天）
  - 组数 / 次数 / 休息时间
- 训练逻辑符合真实健身编排

---

### 4. AI 聊天健身助手
- 多轮对话（支持上下文记忆）
- 自动结合用户身体数据回答
- 支持主动追问（模拟真人教练）

---

### 5. 聊天 → 功能联动
- 用户提问后，AI 可推荐下一步操作：
  - 👉 去生成训练计划
  - 👉 查看饮食建议
  - 👉 重新评估体脂

---

### 前端

- WXML + WXSS + JS

### 后端
- FastAPI
- Python

### AI能力
- Qwen API（通义千问）

## ⭐ 项目亮点

### 1. 用户画像驱动的 AI 回答
- 自动读取用户身高 / 体重 / 年龄
- 强制 AI 进行数值分析（如蛋白质计算）

---

### 2. Prompt 工程优化
- 限制输出 JSON 结构
- 强制动作级训练计划
- 控制 AI 进行具体数值计算

---

### 3. 从“问答”到“产品流程”
- 聊天不仅回答，还能引导用户进入下一步功能
- 实现 AI + 应用流程的结合

---

### 4. 结构化 UI 展示
- 训练计划：动作列表卡片化
- 饮食建议：餐次结构化展示
- 提升可执行性与产品体验
